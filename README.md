**A Meta PyTorch OpenEnv Hackathon Submission**

Fortis L7 is a Reinforcement Learning environment built using Meta's OpenEnv framework. It models a high-stakes, real-world cybersecurity task: acting as an intelligent, autonomous Load Balancer to mitigate sophisticated Layer-7 (Application) DDoS attacks and adversarial LLM bot traffic, while preserving legitimate user access.

---

## 🌍 Real-World Utility (The Domain)
Modern botnets bypass standard Layer-3/4 firewalls by simulating legitimate human behavior. They execute heavy, resource-intensive database queries and scraping operations that exhaust server CPU. Fortis L7 provides a training ground for RL agents to learn the nuanced differences between human traffic and AI-driven anomalies, filling a critical gap in infrastructure security training.

## 🏗️ OpenEnv Architecture & Design
The environment fully implements the OpenEnv specification (`step()`, `reset()`, `state()`) with strictly typed models for actions and observations.

### State Representation (`state()`)
At each step, the agent observes a `TrafficBatch` object containing:
* `server_cpu_load`: Float (0.0 - 1.0)
* `request_logs`: JSON array containing `ip_address`, `user_agent`, `session_duration`, and `payload_signature`.

### Action Space
The agent routes traffic by outputting typed commands:
* `ALLOW_TRAFFIC`: Routes the batch to the main server.
* `BLOCK_IP`: Drops the connection.
* `DEPLOY_CHALLENGE`: Issues a CAPTCHA (imposes a slight latency penalty).
* `INSPECT_PAYLOAD`: Calls an expensive LLM to analyze the request syntax.

---

## 🎯 Task Progression & Graders

To challenge frontier models, Fortis L7 features three difficulty tiers with deterministic and reproducible graders. All rewards are normalized to a `[0.0, 1.0]` scale.

### Task 1: Volumetric Flood (Easy)
* **Objective:** Identify and block IPs sending repetitive, high-frequency requests.
* **Grader:** Purely programmatic. Checks if the agent blocked IPs exceeding rate limits while allowing normal baseline traffic. 

### Task 2: Distrubuted Scraping (Medium)
* **Objective:** Mitigate slow-rate distributed scraping across multiple IPs using spoofed User-Agents.
* **Grader:** Programmatic + Simple Pattern Matching. The agent must utilize the `DEPLOY_CHALLENGE` action to verify ambiguous traffic without dropping human sessions.

### Task 3: The Adversarial LLM Flood (Hard)
* **Objective:** Defeat an attack where payloads are dynamically generated by an LLM to mimic complex, unique human database queries designed to spike CPU load.
* **Grader:** LLM-in-the-loop. The agent must strategically use the `INSPECT_PAYLOAD` tool. The environment utilizes an LLM Grader to evaluate the semantic intent of the payload. The agent is scored on its ability to isolate malicious prompt structures while maintaining a sub-80% `server_cpu_load`.

### 📈 Reward Shaping (Partial Progress)
The reward function provides dense, partial progress signals rather than sparse binary outcomes. 
* **Formula:** `Score = (Legitimate_Requests_Served * 0.6) + (Malicious_Blocked * 0.4) - (CPU_Overhead_Penalty) - (False_Positive_Penalty)`
* *Note: If `server_cpu_load` reaches 1.0, the episode terminates early with a score of 0.0.*

## 🚀 Quick Start & Spec Compliance

This repository is built for reproducibility and clean state management. `openenv validate` passes out of the box.

### Prerequisites
* Python 3.10+
* Docker (for containerized execution)
* OpenEnv CLI

### Installation & Testing
```bash
# Clone the repository
git clone [https://github.com/YourOrg/Fortis-L7.git](https://github.com/YourOrg/Fortis-L7.git)
cd Fortis-L7

# Install dependencies
pip install -r requirements.txt

# Run the baseline inference script to reproduce 0.0-1.0 scores
python scripts/run_baseline.py

# Validate OpenEnv compliance
openenv validate .
